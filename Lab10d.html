CPS530 Lab 10 – Tailwind CSS Framework Webpage

1. Framework I Chose

For this project, I chose the Tailwind CSS framework. It is a utility-first CSS framework that allows fast styling using predefined classes. Tailwind is allowed in this assignment because only Bootstrap and jQuery are restricted.

2. How I Installed the Framework

I installed Tailwind CSS using the simplest method: the CDN link. This method requires no compilation, no Node.js, and no backend configuration. I simply added the following line inside the <head> section: 

<script src="https://cdn.tailwindcss.com"></script>

After adding this line, all Tailwind utility classes became available instantly.

3. How I Built This Page

I created a new GitHub repository, added this index.html file, and enabled GitHub Pages to host it online. Tailwind CSS styling automatically works because the CDN loads the framework directly in the browser.

4. Difficulties I Encountered

  • I initially did not know that GitHub Pages requires enabling "Deploy from Branch" before the site becomes public.
  • I was not familiar with Tailwind class names, and there were many utility classes to learn.
  • Sometimes Tailwind styles did not refresh immediately because of browser caching.

5. How I Solved These Issues

  • I checked GitHub documentation and turned on GitHub Pages correctly.
  • I referred to Tailwind’s official documentation and copied example classes.
  • I force refreshed the browser using Ctrl + Shift + R to clear cache.
